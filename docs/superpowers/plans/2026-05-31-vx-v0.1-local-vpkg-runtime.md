# VX v0.1 Local VPKG Runtime Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the inherited directory-browser default with a project-local `vx` runtime that can discover `vpkg` packages, inspect them with `vx view`, and preview or apply `template` exports and direct `.vxt` files with `vx gen`.

**Architecture:** Keep `cmd/vx` thin and move behavior into focused internal packages: one for project-root discovery, one for `vpkg` manifest/discovery, one for target resolution, one for input loading, one for `view`, and one for generation. Treat `vxt` as the only typed template/input contract source, and keep `vpkg.yaml` limited to package/export metadata plus export layout.

**Tech Stack:** Go 1.25, Cobra, existing `internal/testutil`, YAML/JSON decoding, `github.com/vandordev/vxt`, `just test`, `go test`.
