# VX v0.3 Project Context Go Detection Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add always-on `vx` injected project context for `project.root`, detected Go language, `project.go.module`, and `project.go.module_root` for the v0.3.0 milestone.

**Architecture:** Keep `cmd/vx` thin: CLI commands find the root, get the current working directory, call an internal project-context detector, and pass the result into internal services. Go detection and context-to-template input shaping live in `internal/project`; `internal/view` and `internal/gen` only merge the detected context into the `vxt` input map before validation/planning.

**Tech Stack:** Go 1.25, Cobra, existing `internal/testutil`, `github.com/vandordev/vxt`, `just test`, focused `go test` commands.

## Scope

Implement only injected project context owned by `vx`:

- `project.root`
- `project.language`
- `project.go.module`
- `project.go.module_root`

Do not add new `vxt` directives. Do not redesign `vpkg`, `view`, or `gen` runtime behavior beyond the injection integration points listed below.

func DetectContext(projectRoot string, cwd string) (Context, error) {

- [ ] **Step 3: Implement upward `go.mod` walking from `cwd`**

In `internal/project/context.go`:

- Resolve `projectRoot` and `cwd` to absolute paths.
- If `cwd` points to a file, walk from its parent.
- At each directory from `cwd` upward:
  - stop when the directory is outside `projectRoot`
  - check for `go.mod`
  - use the first valid in-root `go.mod`
- Use `filepath.Rel(absRoot, moduleDir)` for `GoContext.ModuleRoot`.
- Keep `ModuleRoot` as `"."` when `go.mod` is at `projectRoot`.

- [ ] **Step 4: Parse the `module` directive**

Implement a small helper:

```go
func parseGoModule(path string) (string, error)
```

Rules:

- read the file
- trim whitespace per line
- ignore blank lines and `//` comment lines
- find the first line whose first field is `module`
- return the second field
- accept quoted module paths via `strconv.Unquote`
- return empty string with nil error when no module directive exists

Do not add `golang.org/x/mod` for this milestone unless the manual parser becomes demonstrably insufficient.
